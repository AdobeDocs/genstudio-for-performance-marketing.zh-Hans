# 波兰语GenStudio发行说明

**规范目标文件：** [help/user-guide/release-notes.md](../../../help/user-guide/release-notes.md)

**示例：** [示例.md](examples.md)

**正在起草工作流（上游）：** [生成发行说明](../generate-release-notes/SKILL.md)

## 范围（必需）

仅波兰语&#x200B;**用户标识为**&#x200B;新添加的&#x200B;**内容**&#x200B;此轮：

- **在文件**&#x200B;中`help/user-guide/release-notes.md`中单个标题`## YYYY.MM {#latest}`下的`###`子节中。
- **不要**&#x200B;编辑&#x200B;**早期发行说明**（可折叠的`+++`块）下的页面标题、介绍段落、前言或任何文本。
- **除非用户**&#x200B;明确&#x200B;**要求，否则不要**&#x200B;编辑之前`##`个每月块或同一`{#latest}`节中预先存在的`###`。
- 如果不清楚哪些`###`是新的，**询问**&#x200B;或使用&#x200B;**git diff** /在更改文章之前编辑上下文。
- **粘贴的内容：**&#x200B;仅当用户确认它与那些&#x200B;**相同**&#x200B;新子部分匹配时，用户粘贴的Markdown修饰符（不将不相关的粘贴视为作用域内）。

将差值&#x200B;**最小化**：仅使用字词和段落分隔符 — 不会改变文件其他位置的逐行转换。

## 语音和语调

- 针对以下产品发行说明，请以&#x200B;**撰稿人**&#x200B;的身份撰写： **conscentical**、**scannable**&#x200B;和&#x200B;**entived**&#x200B;关于&#x200B;**新值** — 以结果为导向和“对您有哪些好处”，使用强动词和简洁的权益语言。
- **保持准确：**&#x200B;除了该功能提供的内容外，没有其他声明；与Experience League/Adobe的语气匹配（自信而明确，而不是耸人听闻）。
- **通过**&#x200B;特定的&#x200B;**优惠和优惠（营销人员现在可以做什么，可以消除哪些摩擦）来激发兴趣** — 不是空洞的宣传或未经证实的夸大其词。

## 段落规则

- 每个段落： **2-3个句子**。 **一个段落中的句子不能超过三个** — 如果需要更多，请开始一个&#x200B;**新段落**。
- 可选：如果段落在渲染文档中仍以典型行长度运行&#x200B;**到3行**&#x200B;以上，请进一步拆分。

## 不添加

- **过程“如何”**&#x200B;内容：编号步骤、“单击&#x200B;**[!UICONTROL ...]** then...”、完整UI演练或教程用语。 发行说明总结了&#x200B;**提供的内容**&#x200B;和&#x200B;**为什么它很重要**，而不是实践课程。
- 在生成技能上违反[禁止的内容](../generate-release-notes/SKILL.md#prohibited-content)的内容（无Jira密钥、仅限内部的URL、wiki-as-proof等）。

## 保留（请勿在结构上剥离或重写）

- `[!DNL …]`、`[!UICONTROL …]`、`[!BADGE …]`和其他ExL快捷键。
- 现有&#x200B;**相对**&#x200B;文档链接和锚点模式： `[phrase](/help/...)`位于有意义的锚点文本上。
- Beta徽章块与[生成发行说明示例](../generate-release-notes/examples.md)中使用的完全相同。

## 工作流核对清单

1. [ ]确认`## … {#latest}`下的&#x200B;**`###`的**&#x200B;在作用域中（此轮为新）。
2. [ ]对于每个作用域中的`###`，请严格按照[语音和音调](#voice-and-tone)和[段落规则](#paragraph-rules)复制副本。
3. [ ]删除或缩短任何&#x200B;**操作说明**&#x200B;说明；保留&#x200B;**用户结果**。
4. [ ]验证链接和短代码仍然有效；根据[质量检查](#quality-checks)运行内部ID或禁止模式的快速扫描。

## 质量检查

- [ ]仅更改了`{#latest}`下同意的&#x200B;**新** `###`块；存档和旧月份未更改。
- [ ]没有新的Jira样式ID、内部Wiki URL或“查看票证”语言。
- [ ]段为&#x200B;**2-3个句子，每段**（每段最多三句）。
- [ ]副本保持&#x200B;**事实**&#x200B;并与描述的功能一致。
